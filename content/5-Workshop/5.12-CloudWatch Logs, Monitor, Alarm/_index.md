#### Objective

In this step, the team configures **Amazon CloudWatch** to monitor the serverless backend of the RoughLife online system. CloudWatch is used to observe logs from the `RoughLifeRoomApi` Lambda function, inspect API invocations, check request duration, monitor memory usage, and create an alarm when Lambda errors occur.

This step is important for an online multiplayer system because the backend must be observable. The team needs to detect issues when players create rooms, list rooms, save player data, or save match results.

---

#### Role of CloudWatch in the RoughLife System

In the RoughLife online architecture, CloudWatch acts as the observability layer.

CloudWatch helps the team monitor:

- Logs generated by the `RoughLifeRoomApi` Lambda function.
- Recent Lambda invocations.
- Request processing time.
- Billed duration and memory usage.
- Runtime errors or request processing errors.
- Alarms when Lambda errors occur.

The monitoring flow is:

1. The Unity Client calls API Gateway.
2. API Gateway forwards the request to the `RoughLifeRoomApi` Lambda function.
3. Lambda processes the request and automatically writes logs to CloudWatch Logs.
4. CloudWatch displays log events, invocation metrics, and alarms.
5. If Lambda errors occur, CloudWatch Alarm can move to an alert state.

---

#### Implementation Steps

1. Open **AWS Lambda** and select the `RoughLifeRoomApi` function.
2. Go to the **Monitor** tab to check CloudWatch Logs.
3. Open the log group `/aws/lambda/RoughLifeRoomApi`.
4. Inspect the log streams and log events created after Lambda executions.
5. Check the **Recent invocations** table to review duration, billed duration, and memory usage.
6. Create a CloudWatch Alarm for the Lambda `Errors` metric.
7. Configure the alarm condition: trigger when Lambda errors are greater than or equal to 1 within 5 minutes.
8. Confirm that the alarm is created in the CloudWatch Alarms list.

#### Creating a CloudWatch Alarm for Lambda Errors

![Create Lambda Error Alarm](/images/5-Workshop/5.12-CloudWatch Logs, Monitor, Alarm/11_04_CloudWatch_Create_Lambda_Error_Alarm.png)

The image above shows the alarm configuration for the `Errors` metric of the `RoughLifeRoomApi` Lambda function.

Main alarm configuration:

- Namespace: `AWS/Lambda`
- Metric name: `Errors`
- FunctionName: `RoughLifeRoomApi`
- Statistic: `Sum`
- Period: `5 minutes`
- Threshold type: `Static`
- Condition: `Greater/Equal`
- Threshold value: `1`
- Datapoints to alarm: `1 out of 1`

With this configuration, if the Lambda function produces one or more errors within a 5-minute period, the CloudWatch Alarm will enter an alert state. This helps the team detect backend failures early.

#### CloudWatch Alarm Created Evidence

![CloudWatch Alarm Created](/images/5-Workshop/5.12-CloudWatch Logs, Monitor, Alarm/11_05_CloudWatch_Alarm_Created.png)

The image above shows that the `RoughLifeRoomApi-Lambda-Error-Alarm` alarm has been created successfully in CloudWatch Alarms.

The alarm is currently in the **Insufficient data** state because there were not enough new error datapoints at the time of creation. This is normal after creating an alarm or when no Lambda errors have occurred yet.

When real Lambda errors occur and the condition `Errors >= 1` within 5 minutes is met, the alarm will move to the **In alarm** state.

#### Result

After this step, the RoughLife system has a basic monitoring layer for its serverless backend. The `RoughLifeRoomApi` Lambda function successfully writes logs to Amazon CloudWatch Logs. The team can inspect each request, execution duration, memory usage, and possible runtime errors.

In addition, a CloudWatch Alarm was configured to monitor the Lambda `Errors` metric. This allows the system to detect backend issues early when the Room API encounters errors.

Adding CloudWatch makes the RoughLife online architecture more practical because a real online game backend needs observability, error inspection, and alerting during operation.
