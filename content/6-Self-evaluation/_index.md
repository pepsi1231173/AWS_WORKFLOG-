# Self-Assessment

During my internship at **Amazon Web Services Viet Nam Company Limited** in the **Workforce Bootcamp - First Cloud AI Journey** program, from **20/04/2026 to 12/07/2026**, I had the opportunity to learn, practice, and apply both university knowledge and new cloud concepts in a practical environment.

Throughout the internship, I worked on my **RoughLife** project and completed related learning activities such as worklogs, proposal writing, translated blogs, event reports, and workshop documentation. These tasks helped me improve my understanding of AWS services, cloud architecture, technical writing, teamwork, and project-based learning.

## Work Completed

- Maintained weekly worklogs to track learning progress and internship tasks.
- Researched and wrote proposal content for the RoughLife project.
- Translated and summarized AWS and cloud-related blog posts.
- Participated in FCAJ and AWS Study Group events, then documented key takeaways.
- Built workshop content with screenshots, explanations, and step-by-step guidance.
- Improved the internship report website using Hugo and organized content in both English and Vietnamese.

## Skills Improved

### Technical Skills

I improved my ability to read technical documentation, organize project content, and connect AWS services with practical use cases. Through the workshop and proposal sections, I became more familiar with topics such as VPC, S3 access patterns, endpoint architecture, CloudFront, WAF, DevOps automation, AI-assisted workflows, and secure cloud design.

### Documentation and Reporting

Before the internship, I was not very confident in writing long technical reports. During the internship, I practiced writing more structured content, explaining technical steps clearly, and using screenshots to support the reader. This helped me understand that good documentation is not just about recording what was done, but also explaining why each step matters.

### Self-Learning

The program required me to learn many new concepts independently. At first, I needed a lot of time to understand unfamiliar AWS topics, but over time I became better at searching documentation, comparing examples, asking clearer questions, and summarizing what I learned.

### Teamwork and Communication

Working with teammates helped me improve communication and task coordination. I learned to ask for support when needed, share progress more clearly, and listen to feedback from others. I still need to become more confident when communicating outside my immediate group, but this internship helped me make real progress.

## Strengths

- Willing to learn new technologies and improve from feedback.
- Able to document learning progress and organize report content.
- Responsible with assigned tasks and committed to completing the internship report.
- Interested in applying cloud concepts to a real project instead of only studying theory.
- More confident in using AI tools to support research, writing, and technical explanation.

## Areas for Improvement

- Need to improve speed when learning unfamiliar technical concepts.
- Need to be more proactive in asking questions and proposing solutions earlier.
- Need to strengthen communication with people outside the immediate team.
- Need more hands-on practice with AWS services to build deeper technical confidence.
- Need to improve problem-solving by analyzing root causes more carefully before fixing.

## Overall Reflection

This internship helped me understand the gap between studying technology and applying it in a real project. I learned that technical growth requires consistency, patience, documentation, and the ability to receive feedback. Although I still have areas to improve, I believe the internship helped me become more disciplined, more aware of my weaknesses, and better prepared for future cloud and software engineering work.
